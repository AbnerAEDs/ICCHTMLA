Como funciona o sistema de controle de versões Git

O git é um sistema de controle de versão distribuido amplamente utilizado para gerenciar projetos de software. Ele permite rastrear alterações em arquivos e coordenar o trabalho de várias pessoas em um projeto. aqui está uma visão geral de como utilizar o git.

Instalação
Para a instalação do git para seu windows, MacOS e linux você pode baixar a versão apropriada para seu sistema operacional no site oficial do Git No site terá informações importante sobre o git e sua ultima versão para download.
Repositório
Após fazer a instalação de umas das verções do git você terá a opção de criar um repositório. O Repositorio é um local aonde se armazena um arquivo e mostra suas alterações do seu projeto, para iniciar um repositório, vá para o diretorio raiz do seu projeto e utilize o comando git init no terminal.
Adicionar arquivo no repositório
Após iniciar o repositório você precisa adicionar os arquivos do seu projeto ao controle do Git. Use o comando git add "nome do arquivo" para adicionar um arquivo especifico ou "git add ." para adicionar todos os arquivos do diretório atual.
Confirmar alterações
Ao adicionar os arquivos,você precisa confirmar as alterações. Use o comando "git commit -m "mensagem do commit"" para criar um commit com uma mensagem descritiva das alterações feitas. a mensagem do commit é utilizada apenas para dar um nome para alteração feita.
Confirmar Commit
Confirmando as alterações, o usuario precisa utilizar o comando push para enviar as atualização do commit, usando um terminal ou Prompt de comando digite git push após terminar as fases acima.

Para conseguir fazer as ações acima, aqui vai algumas explicações de comandos importantes e auxiliares.

Comandos Descrição
Init: Cria um novo repositório Git vazio no diretório atual.
Clone: Cria uma cópia de um repositório Git existente em um novo diretório. É usado para obter uma cópia local de um repositório remoto.
Branch: Lista, cria ou exclui branches (ramificações) no repositório. Branches são usados para desenvolver recursos isolados um do outro.
Checkout: Alterna para um branch específico ou restaura arquivos do diretório de trabalho.
Status: Exibe o status atual do repositório, mostrando as alterações realizadas nos arquivos.
Diff: Mostra as diferenças entre o conteúdo dos arquivos no diretório de trabalho e as alterações confirmadas (commits) no repositório.
Add: Adiciona alterações no diretório de trabalho ao índice (staging area), preparando-as para o próximo commit.
Commit: Grava as alterações adicionadas ao repositório, criando um novo commit com uma mensagem descritiva.
Push: Envia os commits locais para um repositório remoto especificado. É usado para enviar as alterações para compartilhamento com outros colaboradores.
Pull: Obtém as alterações de um repositório remoto e as mescla com o branch atual.
Revert: Desfaz um commit específico, criando um novo commit com as alterações revertidas.
Merge: Combina as alterações de um branch com outro branch ativo.
Stash: Armazena as alterações atuais em um local temporário, permitindo alternar para outro branch sem precisar fazer um commit das alterações incompletas.
Rm: Remove arquivos do diretório de trabalho e do controle de versão do Git.
Config: Configura as opções do Git, como nome de usuário, endereço de e-mail, entre outros.
Reset: Desfaz as alterações em um arquivo, commit ou branch específico. Pode ser usado para mover o branch atual para um commit anterior.
Remote: Gerencia repositórios remotos, permitindo adicionar, renomear ou remover conexões com repositórios remotos.
Fetch: Obtém as alterações de um repositório remoto sem mesclá-las com o branch atual.
Show: Mostra informações sobre um commit específico, como as alterações realizadas.
Help: Exibe a documentação de ajuda do Git ou informações sobre um comando específico.
Rebase: Move ou combina commits de um branch com outro, reescrevendo a história do projeto.
Blame: Mostra quem modificou cada linha de um arquivo, exibindo o autor e o commit correspondente.
Tag: Marca um ponto específico em um histórico de commits, geralmente usado para marcar versões estáveis do projeto.
